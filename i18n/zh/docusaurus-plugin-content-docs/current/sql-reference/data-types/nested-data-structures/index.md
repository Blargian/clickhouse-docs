# 嵌套

## Nested(name1 Type1, Name2 Type2, ...) {#nestedname1-type1-name2-type2-}

嵌套数据结构就像单元格内的一个表。嵌套数据结构的参数（列名和类型）与 [CREATE TABLE](../../../sql-reference/statements/create/table.md) 查询中的指定方式相同。每个表行可以对应于嵌套数据结构中的任意数量的行。

示例：

```sql
CREATE TABLE test.visits
(
    CounterID UInt32,
    StartDate Date,
    Sign Int8,
    IsNew UInt8,
    VisitID UInt64,
    UserID UInt64,
    ...
    Goals Nested
    (
        ID UInt32,
        Serial UInt32,
        EventTime DateTime,
        Price Int64,
        OrderID String,
        CurrencyID UInt32
    ),
    ...
) ENGINE = CollapsingMergeTree(StartDate, intHash32(UserID), (CounterID, StartDate, intHash32(UserID), VisitID), 8192, Sign)
```

此示例声明了 `Goals` 嵌套数据结构，其中包含有关转化（已达成目标）的数据。'visits' 表中的每一行可以对应于零个或任意数量的转化。

当 [flatten_nested](/operations/settings/settings#flatten_nested) 设置为 `0`（默认情况下不是这样）时，支持任意层级的嵌套。

在大多数情况下，当处理嵌套数据结构时，其列用点分隔的列名进行指定。这些列组成了匹配类型的数组。单个嵌套数据结构的所有列数组具有相同的长度。

示例：

```sql
SELECT
    Goals.ID,
    Goals.EventTime
FROM test.visits
WHERE CounterID = 101500 AND length(Goals.ID) < 5
LIMIT 10
```

将嵌套数据结构视为一组相同长度的多个列数组是最简单的。

SELECT 查询中能够指定整个嵌套数据结构的名称而不是单独列的唯一地方是 ARRAY JOIN 子句。如需更多信息，请参见 "ARRAY JOIN 子句"。示例：

```sql
SELECT
    Goal.ID,
    Goal.EventTime
FROM test.visits
ARRAY JOIN Goals AS Goal
WHERE CounterID = 101500 AND length(Goals.ID) < 5
LIMIT 10
```

您无法对整个嵌套数据结构执行 SELECT 查询。您只能显式列出其组成部分的单独列。

对于 INSERT 查询，您应该分别传递嵌套数据结构的所有组成列数组（就像它们是单独的列数组一样）。在插入过程中，系统会检查它们是否具有相同的长度。

对于 DESCRIBE 查询，嵌套数据结构中的列同样以相同的方式单独列出。

ALTER 查询对嵌套数据结构中的元素有一定的限制。
