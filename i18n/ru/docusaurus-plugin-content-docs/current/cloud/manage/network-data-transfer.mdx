ClickHouse Cloud измеряет переданные данные на вход и выход. 
Это включает любые данные, поступающие в и выходящие из ClickHouse Cloud, а также любые передачи данных внутри региона и между регионами. 
Это использование отслеживается на уровне сервиса. На основе этого использования клиенты несут расходы на передачу данных, которые затем добавляются к их ежемесячному счету.

ClickHouse Cloud взимает плату за:
- Исходящие данные из ClickHouse Cloud в публичный Интернет, включая другие регионы других облачных провайдеров. 
- Исходящие данные в другой регион в том же облачном провайдере.

Плата не взимается за передачу данных внутри региона или использование Private Link/Private Service Connect и передачу данных. 
Тем не менее, мы оставляем за собой право внедрять дополнительные параметры ценообразования на передачу данных, если мы увидим паттерны использования, которые влияют на нашу способность адекватно взимать плату с пользователей.

Расходы на передачу данных варьируются в зависимости от облачного провайдера (CSP) и региона. 
Цены на исходящие данные в публичный интернет основываются только на регионах происхождения. 
Цены на межрегиональные (или кросс-региональные) передачи зависят как от региона происхождения, так и от региона назначения.

**Лучшие практики для минимизации расходов на передачу данных**

Существуют некоторые паттерны, которые стоит учитывать при передаче данных для минимизации расходов на передачу данных.
1. При передаче данных в Clickhouse Cloud или из него используйте сжатие, где это возможно, чтобы минимизировать объем переданных данных и сопутствующие расходы.
2. Имейте в виду, что при выполнении INSERT через нативный протокол с ненаправленными значениями (например, INSERT INTO [TABLE] FROM INFILE [FILE] FORMAT NATIVE) клиенты ClickHouse извлекают метаданные с серверов, чтобы упаковать данные. Если метаданные больше, чем нагрузка INSERT, вы можете неожиданно увидеть больше исходящих данных, чем входящих с точки зрения сервера. Если это недопустимо, рассмотрите возможность инлайнинга данных с помощью синтаксиса VALUES или использования HTTP-протокола.
