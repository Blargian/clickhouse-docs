因为 ClickHouse 是开源的，它得到了不仅来自 ClickHouse 员工，还有社区的许多贡献。这些贡献往往在不同的速度下进行；某些功能可能需要较长的原型阶段或更多的时间来进行充分的社区反馈和迭代，以被视为一般可用（GA）。

由于不确定何时功能会被归类为一般可用，我们将功能划分为两个类别：**Beta** 和 **Experimental**。

**Beta** 功能由 ClickHouse 团队正式支持。 **Experimental** 功能是由 ClickHouse 团队或社区驱动的早期原型，并不被正式支持。

以下部分明确描述了 **Beta** 和 **Experimental** 功能的属性：

## Beta Features {#beta-features}

- 正在积极开发以使其可一般可用（GA）
- 主要已知问题可以在 GitHub 上跟踪
- 功能可能在未来发生变化
- 可能在 ClickHouse Cloud 中启用
- ClickHouse 团队支持 beta 功能

以下功能在 ClickHouse Cloud 中被视为 Beta，并可在 ClickHouse Cloud Services 中使用，即使它们当前可能处于一个 ClickHouse 的 SETTING，名为 ```allow_experimental_*```：

注意：使用 ClickHouse 的 [compatibility](/operations/settings/settings#compatibility) 设置确保您在使用一个最近引入的功能。

## Experimental Features {#experimental-features}

- 可能永远不会成为 GA
- 可能被删除
- 可能引入破坏性更改
- 功能可能在未来发生变化
- 需要明确启用
- ClickHouse 团队 **不支持** 实验性功能
- 可能缺乏重要功能和文档
- 不能在云中启用

请注意：在 ClickHouse Cloud 中，除了以上列为 Beta 的功能之外，不允许启用其他实验性功能。
