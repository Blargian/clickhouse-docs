ClickHouse Cloud 监测传入和传出的数据。 
这包括 ClickHouse Cloud 内外的所有数据，以及任何区域内和跨区域的数据传输。 
此使用情况在服务层级进行跟踪。 根据这些使用情况，客户会产生数据传输费用，这些费用会在他们的每月账单中增加。

ClickHouse Cloud 收取以下费用：
- 从 ClickHouse Cloud 到公共互联网的数据传出，包括到其他云提供商的其他区域。 
- 数据传出到同一云提供商的另一个区域。

区域内的数据传输或私有链接/私有服务连接的使用和数据传输没有费用。 
但是，如果我们看到影响我们适当向用户收费能力的使用模式，我们保留实施额外数据传输定价维度的权利。

数据传输费用因云服务提供商 (CSP) 和区域而异。 
公共互联网数据传出定价仅基于源区域。 
跨区域定价依赖于源区域和目标区域。

**减少数据传输成本的最佳实践**

在传入和传出数据时，有一些模式可以帮助最小化数据传输成本。
1. 在从 ClickHouse Cloud 传入或传出数据时，尽可能使用压缩，以最小化传输的数据量及相关成本。
2. 注意，当通过原生协议进行 INSERT 并使用了非内联值时（例如 INSERT INTO [TABLE] FROM INFILE [FILE] FORMAT NATIVE），ClickHouse 客户端会从服务器拉取元数据以打包数据。如果元数据大于 INSERT 有效负载，你可能会反直觉地看到从服务器的传出量大于传入量。如果这不可接受，考虑使用 VALUES 语法将数据内联或使用 HTTP 协议。
