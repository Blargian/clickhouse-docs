# 時系列分析関数

ClickHouseでの時系列分析は、標準のSQL集約およびウィンドウ関数を使用して行うことができます。  
時系列データを扱うとき、通常は3つの主要なメトリックに遭遇します。

* 時間とともに単調に増加するカウンターメトリック（ページビューや総イベントなど）
* 上下するポイントインタイム測定を表すゲージメトリック（CPU使用率や温度など）
* 観測値をサンプリングし、バケットにカウントするヒストグラム（リクエストの期間や応答サイズなど）

これらのメトリックに対する一般的な分析パターンには、期間間での値の比較、累積トータルの計算、変化率の算定、分布の分析が含まれます。  
これらは、集約の組み合わせ、`sum() OVER`のようなウィンドウ関数、`histogram()`のような専門的な関数を通じてすべて達成できます。

## 期間間の変化 {#time-series-period-over-period-changes}

時系列データを分析する際、時間の経過に伴う値の変化を理解する必要があります。  
これはゲージメトリックとカウンターメトリックの両方にとって重要です。  
[`lagInFrame`](/docs/sql-reference/window-functions/lagInFrame)ウィンドウ関数は、これらの変化を計算するために前の期間の値にアクセスすることを可能にします。

次のクエリは、"Weird Al" YankovicのWikipediaページに対する日ごとのビュー数の変化を計算しています。  
トレンドカラムは、前日と比較してトラフィックが増加したか（正の値）または減少したか（負の値）を示し、活動の異常なスパイクやドロップを特定するのに役立ちます。

```sql
SELECT
    toDate(time) AS day,
    sum(hits) AS h,
    lagInFrame(h) OVER (ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING) AS p,
    h - p AS trend
FROM wikistat
WHERE path = '"Weird_Al"_Yankovic'
GROUP BY ALL
LIMIT 10;
```

## 累積値 {#time-series-cumulative-values}

カウンターメトリックは、自然に時間の経過とともに蓄積されます。  
この累積成長を分析するために、ウィンドウ関数を使用してランニングトータルを計算できます。

次のクエリは、`sum() OVER`句を使用してランニングトータルを作成し、`bar()`関数で成長の視覚的表現を提供しています。

```sql
SELECT
    toDate(time) AS day,
    sum(hits) AS h,
    sum(h) OVER (ROWS BETWEEN UNBOUNDED PRECEDING AND 0 FOLLOWING) AS c,
    bar(c, 0, 50000, 25) AS b
FROM wikistat
WHERE path = '"Weird_Al"_Yankovic'
GROUP BY ALL
ORDER BY day
LIMIT 10;
```

## レート計算 {#time-series-rate-calculations}

時系列データを分析する際、時間単位あたりのイベントのレートを理解することは非常に有用です。  
このクエリは、ページビューのレートを計算するために、1時間あたりの合計を3600秒で割っています。  
視覚的なバーは、活動のピーク時間を特定するのに役立ちます。


```sql
SELECT
    toStartOfHour(time) AS time,
    sum(hits) AS hits,
    round(hits / (60 * 60), 2) AS rate,
    bar(rate * 10, 0, max(rate * 10) OVER (), 25) AS b
FROM wikistat
WHERE path = '"Weird_Al"_Yankovic'
GROUP BY time
LIMIT 10;
```

## ヒストグラム {#time-series-histograms}

時系列データの一般的な使用ケースは、追跡されたイベントに基づいてヒストグラムを構築することです。  
たとえば、総ヒット数が10,000以上のページに基づいて、その分布を理解したい場合です。  
`histogram()`関数を使用して、ビンの数に基づいた適応的なヒストグラムを自動生成できます。

```sql
SELECT
    histogram(10)(hits) AS hist
FROM
(
    SELECT
        path,
        sum(hits) AS hits
    FROM wikistat
    WHERE date(time) = '2015-06-15'
    GROUP BY path
    HAVING hits > 10000
)
FORMAT Vertical;
```

次に、[`arrayJoin()`](/docs/sql-reference/functions/array-join)を使用してデータを整形し、`bar()`で視覚化します。

```sql
WITH histogram(10)(hits) AS hist
SELECT
    round(arrayJoin(hist).1) AS lowerBound,
    round(arrayJoin(hist).2) AS upperBound,
    arrayJoin(hist).3 AS count,
    bar(count, 0, max(count) OVER (), 20) AS b
FROM
(
    SELECT
        path,
        sum(hits) AS hits
    FROM wikistat
    WHERE date(time) = '2015-06-15'
    GROUP BY path
    HAVING hits > 10000
);
```
