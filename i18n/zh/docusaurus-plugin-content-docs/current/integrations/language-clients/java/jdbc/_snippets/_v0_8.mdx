:::note
`clickhouse-jdbc` 实现了标准 JDBC 接口，并使用最新的 java 客户端。如果性能/直接访问至关重要，我们建议直接使用最新的 java 客户端。

## 0.7.x 的变更 {#changes-from-07x}
在 0.8 中，我们尝试让驱动程序更严格地遵循 JDBC 规范，因此移除了一些可能会影响您的功能：

| 旧功能                           | 备注                                                                                                                                                                                                                                                                                                           |
|----------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 事务支持                         | 驱动程序的早期版本仅仅是 **模拟** 了事务支持，这可能导致意外的结果。                                                                                                                                                                                                                                            |
| 响应列重命名                     | `ResultSet` 是可变的 - 为了提高效率，现在变为只读                                                                                                                                                                                                                                                            |
| 多语句 SQL                       | 多语句支持之前仅是 **模拟** 的，现在严格遵循 1:1 的实现                                                                                                                                                                                                                                                            |
| 命名参数                         | 不属于 JDBC 规范的一部分                                                                                                                                                                                                                                                                                       |
| 基于流的 `PreparedStatement`     | 驱动程序的早期版本允许非 JDBC 用法的 `PreparedStatement` - 如果您希望使用这样的选项，我们建议查看 [Java Client](/integrations/language-clients/java/client/client.mdx) 及其 [示例](https://github.com/ClickHouse/clickhouse-java/tree/main/examples/client-v2)。 |

:::note
`Date` 存储时不带时区，而 `DateTime` 存储时带时区。如果不小心，可能会导致意外结果。
:::

## 配置 {#configuration}

**驱动类**: `com.clickhouse.jdbc.ClickHouseDriver`

**URL 语法**: `jdbc:(ch|clickhouse)[:<protocol>]://endpoint1[,endpoint2,...][/<database>][?param1=value1&param2=value2][#tag1,tag2,...]`，例如：

- `jdbc:clickhouse:http://localhost:8123`
- `jdbc:clickhouse:https://localhost:8443?ssl=true`

**连接属性**:

除了标准的 JDBC 属性外，驱动程序还支持底层 [java client](/integrations/language-clients/java/client/client.mdx) 提供的 ClickHouse 特定属性。如果特性不受支持，尽可能的方法将返回 `SQLFeatureNotSupportedException`。其他自定义属性包括：

| 属性                             | 默认   | 描述                                                  |
|----------------------------------|--------|------------------------------------------------------|
| `disable_frameworks_detection`   | `true` | 禁用 User-Agent 的框架检测                           |
| `jdbc_ignore_unsupported_values` | `false`| 抑制 `SQLFeatureNotSupportedException`               |
| `clickhouse.jdbc.v1`             | `false`| 使用旧版 JDBC 实现而不是新版 JDBC                   |
| `default_query_settings`         | `null` | 允许在查询操作中传递默认查询设置                    |

## 支持的数据类型 {#supported-data-types}

JDBC 驱动程序支持与底层 [java client](/integrations/language-clients/java/client/client.mdx) 相同的数据格式。

### 处理日期、时间和时区 {#handling-dates-times-and-timezones}
`java.sql.Date`、`java.sql.Time` 和 `java.sql.Timestamp` 在计算时区时可能会造成复杂性 - 尽管这些类型当然是受支持的，您可能希望考虑使用 [java.time](https://docs.oracle.com/javase/8/docs/api/java/time/package-summary.html) 包。`ZonedDateTime` 和 `OffsetDateTime` 是 `java.sql.Timestamp`、`java.sql.Date` 和 `java.sql.Time` 的很好的替代品。

```java showLineNumbers
Properties info = new Properties();
info.put("user", "default");
info.put("password", "password");
info.put("database", "some_db");

//Creating a connection with DataSource
DataSource dataSource = new DataSource(url, info);
try (Connection conn = dataSource.getConnection()) {
... // do something with the connection
}

//Alternate approach using the DriverManager
try (Connection conn = DriverManager.getConnection(url, info)) {
... // do something with the connection
}
```

该驱动程序使用 [slf4j](https://www.slf4j.org/) 进行日志记录，并将使用 `classpath` 上第一个可用的实现。

### 解决大插入时的 JDBC 超时问题 {#resolving-jdbc-timeout-on-large-inserts}

在 ClickHouse 执行长时间的大插入时，您可能会遇到 JDBC 超时错误，例如：

```plaintext
Caused by: java.sql.SQLException: Read timed out, server myHostname [uri=https://hostname.aws.clickhouse.cloud:8443]
```
这些错误可能会中断数据插入过程并影响系统稳定性。为了解决这个问题，您可能需要调整客户端操作系统中的一些超时设置。

#### Mac OS {#mac-os}

在 Mac OS 上，可以调整以下设置以解决该问题：

- `net.inet.tcp.keepidle`: 60000
- `net.inet.tcp.keepintvl`: 45000
- `net.inet.tcp.keepinit`: 45000
- `net.inet.tcp.keepcnt`: 8
- `net.inet.tcp.always_keepalive`: 1

#### Linux {#linux}

在 Linux 上，仅靠对应的设置可能无法解决该问题。由于 Linux 在处理套接字保持活动设置方面的差异，还需要额外的步骤。请按照以下步骤操作：

1. 在 `/etc/sysctl.conf` 或相关配置文件中调整以下 Linux 内核参数：

 - `net.inet.tcp.keepidle`: 60000
 - `net.inet.tcp.keepintvl`: 45000
 - `net.inet.tcp.keepinit`: 45000
 - `net.inet.tcp.keepcnt`: 8
 - `net.inet.tcp.always_keepalive`: 1
 - `net.ipv4.tcp_keepalive_intvl`: 75
 - `net.ipv4.tcp_keepalive_probes`: 9
 - `net.ipv4.tcp_keepalive_time`: 60（您可能考虑将此值从默认的 300 秒降低）

2. 修改内核参数后，运行以下命令以应用更改：

```shell
sudo sysctl -p
```

设置好这些设置后，您需要确保客户端在套接字上启用保持活动选项：

```java
properties.setProperty("socket_keepalive", "true");
```
