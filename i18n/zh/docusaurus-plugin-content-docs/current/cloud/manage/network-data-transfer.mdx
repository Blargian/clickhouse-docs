ClickHouse Cloud 计量传入和传出数据。  
这包括任何输入和输出到 ClickHouse Cloud 的数据，以及任何区域内和跨区域的数据传输。  
此使用情况在服务级别进行跟踪。根据此使用情况，客户会产生数据传输费用，这些费用会添加到他们的月度账单中。

ClickHouse Cloud 的收费包括：  
- 从 ClickHouse Cloud 到公共互联网的数据传出，包括到其他云服务提供商的其他区域。  
- 到同一云服务提供商的另一个区域的数据传出。

区域内数据传输或 Private Link/Private Service Connect 使用和数据传输不收费。  
但是，如果我们看到使用模式影响我们适当地向用户收费的能力，我们保留实施额外数据传输定价维度的权利。

数据传输费用因云服务提供商 (CSP) 和区域而异。  
公共互联网的传出定价仅基于起源区域。  
区域间（或跨区域）的定价取决于起源区域和目标区域。

**最小化数据传输成本的最佳实践**

在传入和传出数据时，有一些模式需要牢记，以最小化数据传输成本。  
1. 在从 ClickHouse Cloud 传入或传出数据时，尽可能使用压缩，以最小化传输的数据量及相关成本。  
2. 请注意，当使用原生协议进行 INSERT 操作时，使用非内联值（例如：INSERT INTO [TABLE] FROM INFILE [FILE] FORMAT NATIVE），ClickHouse 客户端会从服务器拉取元数据以打包数据。如果元数据大于 INSERT 负载，您可能会在服务器视角下意外地看到传出数据量大于传入数据量。如果这不可接受，请考虑使用 VALUES 语法内联数据或使用 HTTP 协议。
