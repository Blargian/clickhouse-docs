# ClickHouse Cloud 快速入门

使用 ClickHouse 的最快和最简单的方法是在 [ClickHouse Cloud](https://console.clickhouse.cloud) 中创建一个新服务。

<VerticalStepper type="numbered">

## 创建 ClickHouse 服务 {#1-create-a-clickhouse-service}

要在 [ClickHouse Cloud](https://console.clickhouse.cloud) 中创建免费的 ClickHouse 服务，您只需通过完成以下步骤进行注册：

  - 在 [注册页面](https://console.clickhouse.cloud/signUp) 创建一个账号
  - 您可以选择使用电子邮件或通过 Google SSO、Microsoft SSO、AWS Marketplace、Google Cloud 或 Microsoft Azure 注册
  - 如果您使用电子邮件和密码注册，请记得在接下来的 24 小时内通过您在电子邮件中收到的链接验证您的电子邮件地址
  - 使用您刚刚创建的用户名和密码登录

<Image img={signup_page} size="md" alt='选择计划' border/>
<br/>

登录后，ClickHouse Cloud 会启动入门向导，指导您创建新的 ClickHouse 服务。您将首先被要求 [选择一个计划](/cloud/manage/cloud-tiers)：

<Image img={select_plan} size="md" alt='选择计划' border/>
<br/>

:::tip
我们建议大多数工作负载选择 Scale 级别。
有关级别的更多细节可以在 [这里](/cloud/manage/cloud-tiers) 找到。
:::

选择计划时，您需要选择部署第一个服务的所需区域。
具体可用的选项将取决于所选的级别。
在下面的步骤中，我们假设用户选择了推荐的 Scale 级别。

选择您希望部署服务的区域，并为您的新服务命名：

<Image img={createservice1} size="md" alt='新的 ClickHouse 服务' border/>
<br/>

默认情况下，Scale 级别将创建 3 个副本，每个副本配备 4 个 VCPU 和 16 GiB RAM。Scale 级别默认启用 [垂直自动扩展](/manage/scaling#vertical-auto-scaling)。

如果需要，用户可以自定义服务资源，指定副本之间的最小和最大大小。在准备就绪时，选择 `Create service`。

<Image img={scaling_limits} size="md" alt='扩展限制' border/>
<br/>

恭喜！您的 ClickHouse Cloud 服务已启动并运行，入门已完成。继续阅读以了解如何开始接收和查询数据。

## 连接到 ClickHouse {#2-connect-to-clickhouse}
有两种方法可以连接到 ClickHouse：
  - 使用基于网页的 SQL 控制台连接
  - 与您的应用程序连接
<br/>
### 使用 SQL 控制台连接 {#connect-using-sql-console}

为了快速入门，ClickHouse 提供了一个基于网页的 SQL 控制台，您将在完成入门后被重定向到此。

<Image img={createservice8} size="md" alt='SQL 控制台' border/>

创建一个查询标签并输入一个简单的查询，以验证您的连接是否正常工作：

```sql
SHOW databases
```

您应该在列表中看到 4 个数据库，以及您可能添加的任何数据库。

<Image img={show_databases} size="md" alt='SQL 控制台' border/>
<br/>

就这样 - 您准备好开始使用新的 ClickHouse 服务了！

### 与您的应用程序连接 {#connect-with-your-app}

从导航菜单中按连接按钮，将打开一个模态框，提供您服务的凭证，并为您提供如何与您的接口或语言客户端连接的一系列说明。

<Image img={service_connect} size="md" alt='服务连接' border/>
<br/>

如果您看不到您的语言客户端，可能需要查看我们的 [集成列表](/integrations)。

## 添加数据 {#3-add-data}

ClickHouse 与数据更搭配！有多种方法可以添加数据，绝大多数方法都可以在导航菜单中的数据源页面访问。

<Image img={data_sources} size="md" alt='数据源' border/>
<br/>

您可以使用以下方法上传数据：
  - 设置 ClickPipe 从数据源（如 S3、Postgres、Kafka、GCS）开始接收数据
  - 使用 SQL 控制台
  - 使用 ClickHouse 客户端
  - 上传文件 - 接受的格式包括 JSON、CSV 和 TSV
  - 从文件 URL 上传数据

### ClickPipes {#clickpipes}

[ClickPipes](http://clickhouse.com/docs/integrations/clickpipes) 是一个托管集成平台，使从各种来源接收数据变得像点击几个按钮一样简单。ClickPipes 的强大且可扩展的架构旨在应对最苛刻的工作负载，确保一致的性能和可靠性。ClickPipes 可用于长期流式需求或一次性数据加载任务。

<Image img={select_data_source} size="md" alt='选择数据源' border/>
<br/>

### 使用 SQL 控制台添加数据 {#add-data-using-the-sql-console}

与大多数数据库管理系统一样，ClickHouse 将表逻辑上分组到 **数据库** 中。使用 [`CREATE DATABASE`](../../sql-reference/statements/create/database.md) 命令在 ClickHouse 中创建一个新数据库：

```sql
CREATE DATABASE IF NOT EXISTS helloworld
```

运行以下命令以在 `helloworld` 数据库中创建一个名为 `my_first_table` 的表：

```sql
CREATE TABLE helloworld.my_first_table
(
    user_id UInt32,
    message String,
    timestamp DateTime,
    metric Float32
)
ENGINE = MergeTree()
PRIMARY KEY (user_id, timestamp)
```

在上面的示例中，`my_first_table` 是一个 [`MergeTree`](../../engines/table-engines/mergetree-family/mergetree.md) 表，包含四列：

  - `user_id`:  一个 32 位无符号整数 ([UInt32](../../sql-reference/data-types/int-uint.md))
  - `message`: 一个 [String](../../sql-reference/data-types/string.md) 数据类型，替代其他数据库系统中的 `VARCHAR`、`BLOB`、`CLOB` 等类型
  - `timestamp`: 一个 [DateTime](../../sql-reference/data-types/datetime.md) 值，表示一个时间点
  - `metric`: 一个 32 位浮点数 ([Float32](../../sql-reference/data-types/float.md))

:::note 表引擎
表引擎决定：
  - 数据的存储方式和位置
  - 支持哪些查询
  - 数据是否被复制
<br/>
有许多表引擎可以选择，但对于单节点 ClickHouse 服务器上的简单表，[`MergeTree`](/engines/table-engines/mergetree-family/mergetree.md) 是您可能的选择。
:::

#### 主键简要介绍 {#a-brief-intro-to-primary-keys}

在继续之前，了解 ClickHouse 中主键的工作机制非常重要（主键的实现可能会让人感到意外！）：

  - ClickHouse 中的主键对每一行 **_不是唯一的_**

ClickHouse 表的主键决定了数据在写入磁盘时的排序方式。每 8192 行或 10MB 数据（称为 **索引粒度**）在主键索引文件中创建一个条目。这个粒度概念创建了一个 **稀疏索引**，可以轻松地适应内存，粒子代表在 `SELECT` 查询期间处理的最小列数据条带。

可以使用 `PRIMARY KEY` 参数定义主键。如果您定义一个没有指定 `PRIMARY KEY` 的表，则该键将成为在 `ORDER BY` 子句中指定的元组。如果您同时指定了 `PRIMARY KEY` 和 `ORDER BY`，则主键必须是排序顺序的子集。

主键也是排序键，表示为 `(user_id, timestamp)` 的元组。因此，存储在每个列文件中的数据将按 `user_id` 然后 `timestamp` 排序。

有关 ClickHouse 核心概念的深入了解，请参见 ["Core Concepts"](../../managing-data/core-concepts/index.md)。

#### 向表中插入数据 {#insert-data-into-your-table}

您可以使用熟悉的 [`INSERT INTO TABLE`](../../sql-reference/statements/insert-into.md) 命令与 ClickHouse 一起使用，但重要的是要理解，向 [`MergeTree`](/engines/table-engines/mergetree-family/mergetree.md) 表插入每条记录都会在存储中创建一个 **分区**。

:::tip ClickHouse 最佳实践
每批插入大量行 - 成千上万甚至数百万行一次。别担心 - ClickHouse 可以轻松处理这种体量 - 而且它将通过向您的服务发送更少的写入请求来 [为您节省开支](/best-practices/selecting-an-insert-strategy#batch-inserts-if-synchronous)。
:::

<br/>

即使对于一个简单的示例，让我们一次插入多于一行：

```sql
INSERT INTO helloworld.my_first_table (user_id, message, timestamp, metric) VALUES
    (101, 'Hello, ClickHouse!',                                 now(),       -1.0    ),
    (102, 'Insert a lot of rows per batch',                     yesterday(), 1.41421 ),
    (102, 'Sort your data based on your commonly-used queries', today(),     2.718   ),
    (101, 'Granules are the smallest chunks of data read',      now() + 5,   3.14159 )
```

:::note
注意 `timestamp` 列是使用各种 [**Date**](../../sql-reference/data-types/date.md) 和 [**DateTime**](../../sql-reference/data-types/datetime.md) 函数填充的。ClickHouse 提供了数百个有用的函数，您可以 [在 **Functions** 部分查看](/sql-reference/functions/)。
:::

让我们验证它是否正常工作：

```sql
SELECT * FROM helloworld.my_first_table
```

### 使用 ClickHouse 客户端添加数据 {#add-data-using-the-clickhouse-client}

您也可以使用名为 [**clickhouse client**](/interfaces/cli) 的命令行工具连接到您的 ClickHouse Cloud 服务。单击左侧菜单中的 `Connect` 来访问这些详细信息。从对话框中选择下拉菜单中的 `Native`：

<Image img={client_details} size="md" alt='clickhouse client 连接详细信息' border/>
<br/>

1. 安装 [ClickHouse](/interfaces/cli)。

2. 运行命令，替换您的主机名、用户名和密码：

```bash
./clickhouse client --host HOSTNAME.REGION.CSP.clickhouse.cloud \
--secure --port 9440 \
--user default \
--password <password>
```
如果您得到一个微笑的提示，您已准备好执行查询！
```response
:)
```

3. 通过运行以下查询来尝试一下：

<br/>

```sql
SELECT *
FROM helloworld.my_first_table
ORDER BY timestamp
```

注意响应以漂亮的表格格式返回：

```response
┌─user_id─┬─message────────────────────────────────────────────┬───────────timestamp─┬──metric─┐
│     102 │ Insert a lot of rows per batch                     │ 2022-03-21 00:00:00 │ 1.41421 │
│     102 │ Sort your data based on your commonly-used queries │ 2022-03-22 00:00:00 │   2.718 │
│     101 │ Hello, ClickHouse!                                 │ 2022-03-22 14:04:09 │      -1 │
│     101 │ Granules are the smallest chunks of data read      │ 2022-03-22 14:04:14 │ 3.14159 │
└─────────┴────────────────────────────────────────────────────┴─────────────────────┴─────────┘

4 rows in set. Elapsed: 0.008 sec.
```

4. 添加一个 [`FORMAT`](../../sql-reference/statements/select/format.md) 子句以指定 ClickHouse 支持的 [多种输出格式](/interfaces/formats/)：

<br/>

```sql
SELECT *
FROM helloworld.my_first_table
ORDER BY timestamp
FORMAT TabSeparated
```
在上面的查询中，输出以制表符分隔返回：
```response
Query id: 3604df1c-acfd-4117-9c56-f86c69721121

102 Insert a lot of rows per batch      2022-03-21 00:00:00     1.41421
102 Sort your data based on your commonly-used queries  2022-03-22 00:00:00     2.718
101 Hello, ClickHouse!  2022-03-22 14:04:09     -1
101 Granules are the smallest chunks of data read       2022-03-22 14:04:14     3.14159

4 rows in set. Elapsed: 0.005 sec.
```

5. 要退出 `clickhouse client`，请输入 **exit** 命令：

<br/>

```bash
exit
```

### 上传文件 {#upload-a-file}

开始使用数据库时，常见任务之一是插入您已经在文件中的一些数据。我们在网上提供了一些示例数据，代表点击流数据 - 包括用户 ID、访问的 URL 和事件的时间戳。

假设我们在名为 `data.csv` 的 CSV 文件中有以下文本：

```bash title="data.csv"
102,This is data in a file,2022-02-22 10:43:28,123.45
101,It is comma-separated,2022-02-23 00:00:00,456.78
103,Use FORMAT to specify the format,2022-02-21 10:43:30,678.90
```

1. 以下命令将数据插入到 `my_first_table`：

<br/>

```bash
./clickhouse client --host HOSTNAME.REGION.CSP.clickhouse.cloud \
--secure --port 9440 \
--user default \
--password <password> \
--query='INSERT INTO helloworld.my_first_table FORMAT CSV' < data.csv
```

2. 注意，如今在 SQL 控制台查询时，新的行会出现在表中：

<br/>

<Image img={new_rows_from_csv} size="md" alt='来自 CSV 文件的新行' />
<br/>
