ClickHouse Cloud учитывает данные, передаваемые на вход и на выход. 
Это включает любые данные, входящие и выходящие из ClickHouse Cloud, а также любые передачи данных внутри региона и между регионами. 
Это использование отслеживается на уровне сервиса. В зависимости от этого использования клиенты несут расходы на передачу данных, которые затем добавляются к их ежемесячному счету.

ClickHouse Cloud взимает плату за:
- Передачу данных из ClickHouse Cloud в публичный интернет, включая другие регионы других облачных провайдеров.
- Передачу данных в другой регион того же облачного провайдера.

Затраты на передачу данных внутри региона или использование Private Link/Private Service Connect не взимается. 
Тем не менее, мы оставляем за собой право внедрять дополнительные параметры цен на передачу данных, если мы заметим закономерности использования, которые влияют на нашу возможность правильно взимать плату с пользователей.

Расходы на передачу данных варьируются в зависимости от облачного провайдера (CSP) и региона. 
Цены на передачу данных в публичный интернет зависят только от региона отправления. 
Цены на межрегиональную передачу (или передачу между регионами) зависят как от региона отправления, так и от региона назначения.

**Рекомендации по минимизации расходов на передачу данных**

Существует несколько моделей, которые следует учитывать при передаче данных на вход и выход, чтобы минимизировать расходы на передачу данных.
1. При передаче данных в ClickHouse Cloud или из него используйте сжатие, когда это возможно, чтобы минимизировать объем передаваемых данных и связанные с этим расходы.
2. Имейте в виду, что при выполнении INSERT с использованием нативного протокола с неинлайновыми значениями (например, INSERT INTO [TABLE] FROM INFILE [FILE] FORMAT NATIVE) клиенты ClickHouse извлекают метаданные с серверов для упаковки данных. Если метаданные больше, чем полезная нагрузка INSERT, вы можете неудивительно увидеть больше выходных данных, чем входящих с точки зрения сервера. Если это неприемлемо, рассмотрите возможность инлайнинга данных с использованием синтаксиса VALUES или использования HTTP-протокола.

ClickHouse Cloud учитывает межрегиональное использование с учетом уровней, от Уровня 1 до Уровня 4, в зависимости от регионов отправления и назначения. :::
